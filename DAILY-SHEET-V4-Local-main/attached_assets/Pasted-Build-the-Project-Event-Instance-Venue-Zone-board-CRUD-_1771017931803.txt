Goal

Implement the data model + API + frontend screens to support this exact structure:

Project
→ Event (aka Event Series)
→ Event Instance (date-based occurrence)
→ Venue
→ Zones (Default Zone auto-created if none)

All child entities must be scoped to an eventInstanceId and usually via zoneId.

Permissions enforced server-side:

Admin can CRUD everything and invite.

Viewer is read-only.
